Небольшое приложение - Аналог файлового менеджера
Состоит из серверной части на APS.NET core и клиентской части в любом визуальном виде (но + см пункт 2)

UI клиента должен быть функциональным (тратить много времени на дизайн не нужно) (+ см 4)

Используем принципы чистого кода, следим за неймингом и структурой проекта. 
Используем DI. 
Критические участки серверного кода оборачиваем в тесты xUnit (+ см пункт 3).

Реализовать функционал:
- Отображения папок и файлов
- Отображение информация о файлах (размер, время изменения)
- Переходы по папкам
- Возможность отфильтровать, по имени, типу, размеру

Плюсом будет:
1) Возможность регистрации (неважно какой пользователь/пароль) используем JWT токен, все API недоступно без регистрации.
2) Клиентская часть на Angular + TS (внимание на типизацию) ИЛИ Avalonia desktop/(wasm с хостингом на том же сервере)(WPF)
3) Если клиент на Angular - несколько тестов на стороне клиента 
4) Красивый UI\UX будет доп плюсом (функционал важнее)
5) Предусмотреть большую нагрузку - количество файлов > 1000
